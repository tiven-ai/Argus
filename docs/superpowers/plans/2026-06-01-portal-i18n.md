# 门户 i18n Implementation Plan（en / zh-CN / ja）

> **For agentic workers:** REQUIRED SUB-SKILL: 本计划用 superpowers:executing-plans 内联执行（翻译为内容创作，由 controller 直接产出，质量优于委派 subagent）。Steps 用 checkbox（`- [ ]`）跟踪。

**Goal:** 把 VitePress 门户三语化（en / zh-CN / ja）：英文保持唯一真源（`content/` symlink 读 `docs/`），中日译文作为门户层资产放 `apps/docs/zh/`、`apps/docs/ja/`，顶栏出现语言切换。

**Architecture:** `config.ts` 加 VitePress `locales`（root=en 不动；zh/ja 各自 themeConfig 指向 `/zh/`、`/ja/`）。三份英文发布页（首页 hero + 接入指南 + 语义约定）各出 zh、ja 译文。ja 译文加"待母语者校对"提示。

**Tech Stack:** VitePress 1.6.4 i18n（`locales`）+ 现有 symlink/preserveSymlinks/srcExclude/自定义主题。纯内容 + 配置，无 TS/测试代码（验证靠 build + 路由检查 + 浏览器核对）。

**Spec:** `docs/superpowers/specs/2026-06-01-portal-i18n-design.md`

**已实测确认（写计划前在真仓库验证）：** 在 `config.ts` 加 `locales: { root, zh }` + 建 `apps/docs/zh/index.md`、`apps/docs/zh/integration/sending-traces.md` 后 `pnpm --filter @argus/docs build` **绿**；dist 同时产出英文 `/`、`/content/...` 与中文 `/zh/`、`/zh/integration/...`；语言标签（简体中文）出现在产物中。symlink content/ 的 root locale 与常规 zh/ 目录并存无冲突。

**命令约定：** 构建 `pnpm --filter @argus/docs build`；dev `pnpm --filter @argus/docs dev`。提交遵循 commitlint。预提交钩子格式化 `*.{md,json,...}`——中文全角标点会被保留。
